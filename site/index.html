01 — What it is

A low-level framed message bus.
Not an HTTP server. Not a gRPC clone.

A reusable transport you embed in your own services — it moves framed messages fast and gets out of the way. You bring the payload bytes; highperf handles framing, multiplexing, and the event loop.

/ reactor

Portable & lock-free

One event loop per thread (shared-nothing). Each connection is pinned to one loop, so the hot path needs no locks. A single seam abstracts epoll (Linux) and kqueue (macOS).

/ api

Callbacks + coroutines

A proven callback core (muduo-lineage) with an optional C++20 coroutine layer on top — co_await a request, resumed on its owning loop. Use whichever fits.

/ messaging

Layered semantics

A framed-message core, with request/response (correlated, multiplexed) and pub/sub (topic fan-out) built on top. Peer-to-peer; a broker is an app, not baked in.

02 — Architecture

Six layers, bottom to top.

Every layer builds only on the one beneath it. All platform #ifdefs live in the bottom layer — nothing above ever sees a raw syscall flag.

03 — Wire protocol

A 16-byte frame header.

little-endian · on_wire = 8 + total_len
0123456789101112131415
magic0..1 · resync
ver2
flags3
total_len4..7 · u32
type8
ext_len9..10 · u16
rsv11
payload_codec12..15 · app tag
fixed header fields length / routing then: type-specific ext (correlation_id / topic) + payload
